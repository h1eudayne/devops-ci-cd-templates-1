# 13. Amazon RDS Option Groups (Cấu hình tính năng & Bảo mật)

Bên cạnh **Parameter Groups** dùng để cấu hình các tham số hoạt động của Database Engine, Amazon RDS còn cung cấp một cơ chế cấu hình bổ sung rất quan trọng là **Option Groups**. Cơ chế này cho phép bạn kích hoạt và quản lý các tính năng nâng cao (options) của Database Engine, đặc biệt là các tính năng liên quan đến bảo mật (Security), kiểm toán (Audit), mã hóa và các tiện ích mở rộng của hệ thống.

---

## I. Khái niệm và Vai trò của Option Groups

* **Bản chất**: **Option Group** là một tập hợp các tính năng bổ sung (options) được tích hợp sẵn trong Database Engine nhưng mặc định chưa được kích hoạt.
* **Sự khác biệt với Parameter Groups**:
  * **Parameter Groups**: Thay đổi các biến cấu hình có sẵn của DB Engine (ví dụ: `max_connections`, `time_zone`...).
  * **Option Groups**: Kích hoạt/Cài đặt các tính năng hoàn toàn mới hoặc các plugin phụ trợ vào Database (ví dụ: kích hoạt tính năng kiểm toán truy cập, mã hóa dữ liệu TDE, tích hợp xác thực Active Directory...).

---

## II. Phân loại Option Groups: Default vs Custom

Tương tự như Parameter Groups, Option Groups cũng được chia thành hai loại:

1. **Default Option Group (Mặc định)**:
   * Được tự động tạo và gán cho DB Instance khi khởi tạo nếu người dùng không chọn Option Group tùy chỉnh nào.
   * Chứa các tính năng mặc định tối thiểu của Database Engine và **không thể chỉnh sửa**.
2. **Custom Option Group (Tùy chỉnh)**:
   * Được tạo ra để kích hoạt và cấu hình các tùy chọn nâng cao phù hợp với yêu cầu hệ thống.
   * Để sử dụng, bạn tạo mới một Custom Option Group tương ứng với Database Engine và phiên bản (version) đang dùng, thêm các options mong muốn, cấu hình các tham số con bên trong option đó, sau đó áp dụng (apply) Custom Option Group cho DB Instance.

---

## III. Các ví dụ thực tế về Option Groups phổ biến

Option Groups thường được sử dụng phổ biến nhất để kích hoạt các plugin kiểm toán an ninh (Database Auditing) như **MariaDB/MySQL Audit Plugin**, cụ thể có các tham số quan trọng sau:

| Tên tùy chọn (Option Setting) | Ý nghĩa / Công dụng | Giá trị mẫu / Định dạng |
|---|---|---|
| **`SERVER_AUDIT_LOGGING`** | Bật hoặc tắt chức năng ghi nhật ký kiểm toán (Audit Logging). | `0` (Tắt) hoặc `1` (Bật) |
| **`SERVER_AUDIT_EVENTS`** | Xác định các loại hành vi/sự kiện sẽ được ghi lại vào log file. | Các giá trị: `CONNECT`, `QUERY`, `QUERY_DDL` (Data Definition Language), `QUERY_DML` (Data Manipulation Language), `QUERY_DCL` (Data Control Language) |
| **`SERVER_AUDIT_INCL_USERS`** | Chỉ định danh sách cụ thể các Database Users sẽ bị theo dõi nhật ký. | Ví dụ: `admin, application_user` (nếu để trống sẽ theo dõi tất cả). |
| **`SERVER_AUDIT_EXCL_USERS`** | Danh sách các Database Users được ngoại lệ, không ghi log (thường áp dụng cho các User hệ thống hoặc User giám sát tự động để tránh tràn bộ nhớ log). | Ví dụ: `rdsadmin, monitoring_user` |

### Một số Option khác cho các Engine khác:
* **TDE (Transparent Data Encryption)**: Kích hoạt mã hóa dữ liệu tại chỗ (mã hóa ổ đĩa và tệp tin dữ liệu) ở cấp độ Database Engine đối với SQL Server hoặc Oracle.
* **Xác thực Kerberos (Kerberos Authentication)**: Cho phép tích hợp Database trực tiếp với dịch vụ Microsoft Active Directory để quản lý định danh tập trung.
* **Oracle Statspack**: Tiện ích thu thập, lưu trữ và hiển thị dữ liệu hiệu năng của Oracle DB.

---

## IV. Tác động của việc áp dụng Option Groups

* **Immediate vs Maintenance Window**:
  * Một số option khi thêm hoặc chỉnh sửa yêu cầu khởi động lại database hoặc có thể mất thời gian thiết lập hạ tầng bổ sung.
  * AWS RDS hỗ trợ lựa chọn áp dụng ngay lập tức (**Apply Immediately**) hoặc đợi đến khung giờ bảo trì định kỳ (**Maintenance Window**) tiếp theo của hệ thống.
* **Ràng buộc tương thích**: Option Group gắn liền với Engine và Phiên bản Database cụ thể (ví dụ: Option Group cho MySQL 8.0 không thể áp dụng cho MySQL 5.7).
